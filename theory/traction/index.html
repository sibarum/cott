1. Preface
1.1 Motivation

At its core, algebra is the symbolic manipulation of expressions subject to declared invariants. Valid manipulations preserve declared invariants while revealing structure.

We call an algebra maximally tractable if it admits every transformation shown to preserve all declared invariants of an expression. Projection, when defined, must map invariant-preserving equivalent expressions to equal projections.

We call an algebra consistent if it forbids manipulations that would invalidate those invariants.

A traction is an algebra designed to be both consistent and maximally tractable: it permits all invariant-preserving transformations and forbids those that violate them.

1.2 Operational Type Extensions

To achieve an exception-free algebra, tractions define new algebraic structures and explicitly specify how symbolic expressions behave under admissible operations, unlike classical algebra, where operations are implicitly assumed to reduce to a single numeric value. Tractions remove this restriction: operations are not required to evaluate numerically. Instead, the unevaluated expression serves as the canonical representative of a distinct algebraic element.

This shift in perspective has an immediate consequence: classical types are defined as sets of numeric elements, not unevaluated expressions. To accommodate symbolic elements, tractions adopt an operational type extension: types are defined by their numeric members and by closure under all admissible operations. In this way, numeric and symbolic elements coexist seamlessly within the same algebraic framework.

Crucially, undefinedness no longer exists; an expression simply does not project to a single numeric value.

Traction Theory is an implementation of Constructive Operational Type Theory. Traction algebra is not an extension of ℝ or ℂ. While it can be helpful to think of classical types as an analogy, this is technically problematic: zero is redefined as non-absorbing and information-preserving, and its reciprocal is unconventional, requiring the underlying type theory itself to be adjusted. For more information, see sibarum.github.io/cott.

Wheels partially implement traction ideas: for example, the zero reciprocal ω restores tractability at singularities, but the introduction of the nullity element ⊥ causes structural collapse rather than totality.

The goal of traction theory is to provide a structure that never collapses, respects all universal invariants, and preserves all symbolic information.

2. Tractions

Tractions are a wheel-like algebra in which the additive identity 0 and its multiplicative reciprocal ω are explicitly treated as valid elements, ensuring the system is exception-free and maximally tractable. Addition forms a closed abelian group. Multiplication is nearly abelian, though additional constraints on the identities introduce subtle deviations. Equality is structural rather than purely equational, comparing the forms of expressions rather than just their numeric evaluation.

2.1 Additive Inverse

For every term t, there exists a corresponding additive inverse, written

–t

Negation is a unary operation satisfying the involution law:

–(–t)=t

Negation is purely structural: forming –t does not evaluate or simplify t; it constructs a new symbolic term.

2.2 Addition

Addition is a total binary operation on terms. For all terms a,b,c:

If

a+b=c

then the recovery laws hold:

c+(–a)=b,
c+(–b)=a.

Addition further satisfies associativity and commutativity:

(a+b)+c=a+(b+c)
a+b=b+a
2.3 Additive Identity

There exists a term 0 such that for every term t:

t+(–t)=0.

From the recovery laws, it follows that

t+0=t,

so 0 acts as an additive identity.

2.4 Multiplicative Inverse

For every term t, there exists a corresponding multiplicative inverse, written

t^–1

This inverse is defined structurally and satisfies the involution law:

(t^–1)^–1=t

Forming t^–1 does not evaluate t; it constructs a new term.

2.5 Multiplication

Multiplication is a total binary operation on terms. For all terms a,b,c:

If

a·b=c

then the recovery laws hold:

c·a^–1=b,
c·b^–1=a

Multiplication further satisfies associativity and commutativity:

(a·b)·c=a·(b·c)
a·b=b·a
2.6 Multiplicative Identity

There exists a term 1 such that for every term t,

t·t^–1=1.

From the recovery laws it follows that

t·1=t.

The inverse of the multiplicative identity is the multiplicative identity.

1^–1=1.
2.7 Zero and ω

The behavior of zero in a traction is not the familiar absorbing element.

Zero does not cause erasure when within a multiplicative association with any other terms.

0+(–0)=0

Zero has a multiplicative inverse, for which we give a convenient alias:

0^–1=ω

Zero and its inverse multiply to the multiplicative identity:

0·ω=1

The additive identity is unique.

2.8 Binding Law

If x ≠ ω then 0 · x is a bound term written 0 x that does not reassociate into additive form.

If x ≠ 0 then ω · x is a bound term written ω x that does not reassociate into additive form.

---
A traction is a totalized, two-group algebra with a distinguished inverse pair (0, ω) and boundary-sensitive reassociation rules.
