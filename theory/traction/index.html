1. Preface
1.1 Motivation

At its core, algebra is the symbolic manipulation of expressions subject to logical invariance. Valid manipulations preserve truth while revealing structure. We call an algebra maximally tractable if it admits every manipulation that preserves the invariants of an expression, allowing structure to be extracted without introducing exceptional cases or undefined behavior. We call an algebra consistent if it forbids manipulations that would invalidate those invariants, ensuring that symbolic transformations do not alter semantic truth. A traction is an algebra designed to be both consistent and maximally tractable: it permits all invariant-preserving transformations while prohibiting those that would break correctness.

1.2 Operational Type Extensions

To achieve an exception-free algebra, tractions define not only new algebraic structures but also new semantics. In classical algebra, operations are implicitly assumed to simplify to a single numeric element - a "number" with digit-wise representation. Tractions remove this restriction: operations are not required to evaluate to a numeric value. Instead, the unevaluated expression itself serves as the canonical symbolic representative of a distinct algebraic element.

This shift in perspective carries an immediate consequence. Classical types are defined as sets of numeric elements, not unevaluated expressions. To accommodate symbolic elements, tractions adopt an operational type extension, in which types are defined not only by their numeric members but also by closure under all admissible operations. In this way, numbers and symbolic expressions coexist seamlessly within the same algebraic framework.

Crucially, undefinedness is no longer a possibility - it is simply an expression that fails to project to a single, exact, numeric value.

Traction Theory is an implementation of Constructive Operational Type Theory. Traction algebra is not an extension of ℝ or ℂ. It may still be helpful to think of them as classic type extensions, but this is technically problematic due to the redefinition and unconventional treatment of zero (non-absorbing, information-preserving) and its reciprocal. This requires the underlying type theory itself to be redefined. Please visit sibarum.github.io/cott for more information about these theories.

Wheels have some elements of tractions, such as a zero reciprocal ω to restore tractability at the singularity, but then collapses again with the introduction of the nullity element ⊥. This doesn't restore totality, but rather converts it into zero-like structural collapse.

The goal of traction theory is to provide a structure that never collapses, respects all universal invariants, and remembers everything else.

2. Tractions
Tractions are a wheel-like algebra wherein the additive identity 0 and its multiplicative reciprocal ω are fully included, ensuring the system is exception-free and maximally tractable. Addition is a closed abelian group. Multiplication is nearly abelian, if not for additional binding constraints on the identities. Equality is structural, not purely equational.
2.1 Additive Inverse

For every term t, there exists a term called its additive inverse, written:

– t

Negation is a unary operation satisfying the involution law

– ( – t ) = t

Negation is purely structural: forming – t does not evaluate or simplify t; it constructs a new term.

2.2 Addition

Addition is a total binary operation on terms. For all terms a, b, c,

a + b = c

implies the recovery laws

c + ( - a ) = b,
c + ( - b ) = a.

Addition further satisfies associativity and commutativity:

( a + b ) + c = a + ( b + c )
a + b = b + a
2.3 Additive Identity

There exists a term 0 such that for every term t,

t + ( – t ) = 0.

From the recovery laws, it follows that

t + 0 = t,

so 0 acts as an additive identity.

2.4 Multiplicative Inverse

For every term t, there exists a term called its multiplicative inverse, written

t ^ –1

This inverse is defined structurally and satisfies the involution law

( t ^ –1 ) ^ –1 = t

Forming t^– 1 does not evaluate t; it constructs a new term.

2.5 Multiplication

Multiplication is a total binary operation. For all terms a, b, c,

a · b = c

implies the recovery laws

c · a ^ –1 = b,
c · b ^ –1 = a.

Multiplication is associative and commutative:

( a · b ) · c = a · ( b · c )
a · b = b · a
2.6 Multiplicative Identity

There exists a term 1 such that for every term t,

t · t ^ –1 = 1.

From the recovery laws it follows that

t · 1 = t.

The inverse of the multiplicative identity is the multiplicative identity.

1^–1 = 1.
2.7 Zero and ω

The behavior of zero in a traction is not the familiar absorbing element.

Zero does not cause erasure when within a multiplicative association with any other terms.

0 + ( – 0 ) = 0

Zero has a multiplicative inverse, for which we give a convenient alias:

0 ^ –1 = ω

Zero and its inverse multiply to the multiplicative identity:

0 · ω = 1

The additive identity is unique.

2.8 Binding Law

If x ≠ ω then 0 · x is a bound term written 0 x that does not reassociate into additive form.

If x ≠ 0 then ω · x is a bound term written ω x that does not reassociate into additive form.

A traction is a totalized, two-group algebra with a distinguished inverse pair (0, ω) and boundary-sensitive reassociation rules.
